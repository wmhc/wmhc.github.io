Java多线程

2022-07-10 Comments

启动一个线程

进程（Processor）
线程（Thread）
一个进程里有多个线程
Runnable接口 ：
创建类Battle，实现Runnable接口
启动的时候，首先创建一个Battle对象，然后再根据该battle对象创建一个线程对象，并启动，线程控制者进程的执行。

Battle battle1 = new Battle(gareen,teemo);
new Thread(battle1).start();
——— 六个状态 ：开启，运行中，阻塞，等待，超时等待，结束———

多线程的三种方法：
1.继承Thread

2.实现Runnable接口

因为java只支持单继承，所以Runnable应运而生，最好是使用接口，而且存放位置不一样，一个存放在Thread子类当中，一个存放在Runnable接口子类当中。

3.匿名类创建。

常见线程方法

1.sleep暂停线程
2.join加入临时线程，主线程会在加入的临时线程结束后才启动
3.setPriority设置线程优先级

可重入锁，就是被一个线程获取到后再获取一次不会被死锁
同步代码块，对象放什么都行，最好是Object，不用另外创建
synchronized(对象/this/类名.class){
中间放需要同步的代码
}
同步函数所用的锁是this。
静态同步函数所用的锁是class，因为静态方法中不可以定义this。静态进内存，内存中没有本类对象，但是一定有该类对应的字节码对象，类名.class,该对象的类是class。

synchronized和lock锁的区别：
1.synchronized是一个关键字，而lock的是一个类方法；
2，sychronized是非公平锁，lock是可以手动设置是否公平
3，synchronized锁是自动释放，lock需要手动释放；
4，synchronized如果阻塞的话是等待是死等，lock不一定，有一个trywait方法
5.synchronized不可以中断，也不可以获取锁的状态，lock可以判断锁的状态，synchronized只适合少量代码。

public class TestThread {

public static void main(String[] args) {
    List<Integer> list1 = new ArrayList<>();
    List<Integer> list2 = Collections.synchronizedList(list1);
}

}
