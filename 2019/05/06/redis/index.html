redis学习笔记

2019-05-06

1高性能键值对数据库，支持的键值数据类型有：

字符串类型。列表类型。有序集合类型。散列类型。*集合类型。读入速度：每秒11w次上下，写入速度每秒8w次

2应用场景：

缓存。聊天在线好友列表。人物队列。网站访问统计。应用排行榜。过过期数据处理，*分布式集群架构中的session分离

3redis安装

下载解压后先make编译
install安装
修改redis.conf文件
把redis.conf放入etc文件中
后台启动 redis—server 。/etc/redis.conf
然后redis-cli连接
到此安装成功

redis数据类型

string 添加：set 获取： get
删除：del
增加一个数字： incr
减少一个：decr
增加制定数字： incrby
减少指定数字：decrby
连接字符串：append
可以存储的最大value512mb

hash 可以存储多个键值对
添加 hset （hashname key value） 获取： hget
设置多个键值对hmset
获取多个属性值 hmget
获取所有属性值 hgetall
删除一个 hdel（hashname key）
del删除整个集合（hashname）
增加数字：hincrby（hashname key）
判断属性是否存在：hexists（hashname key）
获取属性数量 hlen
获得所有key hkeys
获得所有值 hvals

list 链表

ArrayList使用数组方式 LinkedList使用双向链表方式
双向链表增加数据 双向链表删除数据
存储命令：
左侧添加：lpush（listname a b c） 右侧rpush
查看列表 lrange （listname start（0） end（-1））

lrem（list name 数字（正数从头负数尾部0全部） 想要删除的value）
lset （listname 想要第几个插入的数字 value）
linsert（listname before或者after value）再之前或之后插入
rpoplpush（name1 name2 ）取出1的尾部压入2的头部

set：和list不同，不允许出现重复元素，多次添加只保留一个

redis特性

可以包含多个数据库
最多可以连接16个，0-15
select选择数据库
move key 数据库数字（0-15）
multi exec discard
multi 开启事物相当于start
exec提交 commit
discard回滚 rollback

redis持久化

RDB默认方式：指定时间写入一次
AOF：日志的形式记录，

RDB优势：
容易恢复，启动效率高
可能会丢失数据，子进程方式存储如数据过大容易停止几秒
save配置 150行
AOF
安全性高，美秒同步每修改同步
文件大，效率低一点
appendonly yes 540行
flushall 清空数据库
